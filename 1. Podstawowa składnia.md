## ℹ️ Składnia i zmienne

PHP używa prostych instrukcji kończonych średnikiem `;`. Zmienne zaczynają się od znaku dolara `$`, a typy są dynamiczne, co oznacza, że PHP sam dopasowuje typ wartości.

```php
<?php
// Zmienne i typy
$name = "Anna"; // String
$age = 30; // Int
$price = 12.99; // Float
$isActive = true; // Bool

echo "Cześć, $name! Masz $age lat.";
```

```php
<?php
// Operatory matematyczne i przypisania
$a = 10;
$b = 3;

$sum = $a + $b;      // dodawanie
$diff = $a - $b;     // odejmowanie
$product = $a * $b;  // mnożenie
$quotient = $a / $b; // dzielenie
$mod = $a % $b;      // reszta z dzielenia

$a += 5; // dodaj i przypisz (a = a + 5)
$b *= 2; // pomnóż i przypisz (b = b * 2)

echo $sum; // 13
```

Ciągi znaków to jeden z najważniejszych typów danych w PHP. Poniżej znajduje się tabela przydatnych funkcji do pracy z tekstem:

| Funkcja | Opis |
|---------|------|
| `strlen()` | Zwraca długość ciągu |
| `strtoupper()` | Zmienia na WIELKIE litery |
| `strtolower()` | Zmienia na małe litery |
| `strpos()` | Znajduje pozycję podciągu |
| `str_replace()` | Zamienia tekst |
| `substr()` | Wyodrębnia fragment ciągu |
| `str_repeat()` | Powtarza ciąg N razy |
| `explode()` | Dzieli ciąg na tablicę |
| `implode()` | Łączy tablicę w ciąg |
| `trim()` | Usuwa białe znaki z obu stron |


#### Przykłady:
```php
<?php
$text = "Hello World";

// Długość ciągu
echo strlen($text); // 11

// Konwersja wielkości liter
echo strtoupper($text);   // HELLO WORLD
echo strtolower($text);   // hello world

// Szukanie i zamiana
echo strpos($text, "World");        // 6 (pozycja)
echo str_replace("World", "PHP", $text); // Hello PHP
echo str_repeat("Ha", 3);           // HaHaHa

// Wyodrębnianie fragmentów
echo substr($text, 0, 5);   // Hello (litery od 0 do 5)
echo substr($text, 6);      // World (litery od 6)

// Dzielenie i łączenie
$words = explode(" ", $text); // ["Hello", "World"]
echo implode("-", $words);    // Hello-World

// Usuwanie białych znaków
echo trim("  tekst  ");       // tekst
```

## ℹ️ Instrukcje warunkowe 

Warunek to sprawdzenie, czy coś jest prawdziwe czy fałszywe. Jeśli warunek jest prawdziwy, program wykonuje jedną część kodu, a jeśli fałszywy, może wykonać inną.

> Instrukcje warunkowe kontrolują przepływ programu. Najczęściej używane są `if`, `else`, `elseif` oraz `switch`.

#### Przykłady:
```php
<?php
$score = 75;
if ($score >= 90) {
    echo "Ocena: celujący";
} elseif ($score >= 75) {
    echo "Ocena: dobry";
} else {
    echo "Ocena: poprawny";
}
```

```php
<?php
$day = 3;
switch ($day) {
    case 1:
        echo "Poniedziałek";
        break;
    case 2:
        echo "Wtorek";
        break;
    default:
        echo "Inny dzień";
}
```

## ℹ️ Pętle

Pętle pozwalają powtarzać fragment kodu tak długo, jak trzeba. To jak powiedzenie: "Zrób to jeszcze raz".

- `for` – zaczyna od pewnej liczby, powtarza aż do osiągnięcia końca.
- `while` – mówi "rób to, dopóki warunek jest prawdziwy".
- `do-while` – robi raz, a potem sprawdza czy ma powtórzyć.
- `foreach` – bierze po kolei każdy element z listy i robi to samo dla każdego.

> Pętle pozwalają powtarzać fragment kodu. W PHP najczęściej używa się `for`, `while`, `do-while` i `foreach`.

#### Przykłady:
```php
<?php
for ($i = 1; $i <= 5; $i++) {
    echo $i . " ";
}
```

```php
<?php
$number = 1;
while ($number <= 5) {
    echo $number . " ";
    $number++;
}
```

```php
<?php
$fruits = ["jabłko", "banan", "gruszka"];
foreach ($fruits as $fruit) {
    echo $fruit . "\n";
}
```

## ℹ️ Funkcje

Funkcja to kawałek kodu, który wykonuje zadanie. Argument to dane, które podajesz funkcji, aby wiedziała, co ma przetworzyć. Zwrócona wartość to wynik, który funkcja oddaje po wykonaniu.

> Funkcje grupują kod w moduły, które można wywoływać wielokrotnie. Funkcja może przyjmować argumenty i zwracać wartość.

#### Przykłady:
```php
<?php
function greet($name) {
    return "Cześć, $name!"; // wartość zwracana

```

```php
<?php
function add($a, $b) {
    return $a + $b;
}
$result = add(5, 7);
echo $result;
```

## ℹ️ Ciasteczka

Ciasteczka (cookies) to małe pliki przechowywane na komputerze użytkownika. Służą do zapamiętania informacji o użytkowniku między odwiedzinami.

> Ciasteczka pozwalają przechowywać dane na kliencie. Są wysyłane wraz z każdym żądaniem HTTP.

#### Przykłady:
```php
<?php
// Ustawienie ciasteczka
setcookie("username", "Jan", time() + (86400 * 30), "/"); // 30 dni

// Odczytanie ciasteczka
if (isset($_COOKIE["username"])) {
    echo "Witaj, " . $_COOKIE["username"];
}

// Usunięcie ciasteczka
setcookie("username", "", time() - 3600, "/");
```

## ℹ️ Sesje

Sesje pozwalają przechowywać informacje o użytkowniku na serwerze przez czas jego wizyty. W przeciwieństwie do ciasteczek, <ins>dane sesji nie są wysyłane do przeglądarki</ins>.

> Sesje są bezpieczniejsze niż ciasteczka do przechowywania wrażliwych danych. Każdy użytkownik ma unikalny identyfikator sesji.

#### Przykłady:
```php
<?php
session_start(); // Musi być na począku pliku

// Ustawienie zmiennej sesji
$_SESSION["user_id"] = 123;
$_SESSION["username"] = "Anna";

// Odczytanie zmiennej sesji
if (isset($_SESSION["username"])) {
    echo "Zalogowany jako: " . $_SESSION["username"];
}

// Usunięcie zmiennej sesji
unset($_SESSION["username"]);

// Zniszczenie całej sesji
session_destroy();
```

## ℹ️ Wyjątki

Wyjątki pozwalają obsługiwać błędy w kontrolowany sposób. Zamiast crashować program, możemy "złapać" błąd i obsłużyć go w specjalny sposób. Istnieją również domyślne wyjątki wbudowane w język PHP.

> Obsługa wyjątków (`try-catch`) pozwala na bezpieczne wykonanie potencjalnie problematycznego kodu.

| Wyjątek | Opis |
|---------|------|
| `DivisionByZeroError` | Dzielenie przez zero |
| `InvalidArgumentException` | Nieprawidłowy argument funkcji |
| `OutOfRangeException` | Wartość poza dozwolonym zakresem |
| `RuntimeException` | Błąd wykonania w czasie działania |
| `LogicException` | Błąd logiki programu |
| `TypeError` | Nieprawidłowy typ danych |

#### Przykłady:
```php
<?php
try {
    $result = 10 / 0; // To spowoduje błąd
} catch (DivisionByZeroError $e) {
    echo "Błąd: " . $e->getMessage();
}
```

```php
<?php
try {
    if ($age < 0) {
        throw new Exception("Wiek nie może być ujemny!");
    }
    echo "Wiek: $age";
} catch (Exception $e) {
    echo "Złapano wyjątek: " . $e->getMessage();
} finally {
    echo "Koniec sprawdzania wieku";
}
```

| Składnik | Opis |
|----------|------|
| `try` | Kod, który może spowodować błąd |
| `catch` | Obsługa błędu/wyjątku |
| `finally` | Kod, który wykonuje się zawsze |
| `throw` | Rzucenie wyjątku |
| `getMessage()` | Pobranie komunikatu błędu |
| `getCode()` | Pobranie kodu błędu |

## ℹ️ Obsługa plików

Obsługa plików pozwala na czytanie, pisanie i modyfikowanie plików na serwerze.

| Funkcja | Opis |
|---------|------|
| `fopen()` | Otwiera plik |
| `fread()` | Czyta zawartość pliku |
| `fgets()` | Czyta linię |
| `fwrite()` / `fputs()` | Pisze do pliku |

| `fclose()` | Zamyka plik |

| `file_put_contents()` | Zapisuje zawartość |
| `file_exists()` | Sprawdza, czy plik istnieje |
| `unlink()` | Usuwa plik |

#### Przykłady:

<?php
// Czytanie
if (file_exists("dane.txt")) {
    $content = file_get_contents("dane.txt");

$data = "Nowe dane\n";

// Dołączenie
file_put_contents("dane.txt", "Dodatkowe dane\n", FILE_APPEND);

// Usunięcie
if (file_exists("dane.txt")) {
    unlink("dane.txt");
}

```php
<?php
// Otwarcie i czytanie pliku linia po linii
$file = fopen("plik.txt", "r");
if ($file) {
    while (!feof($file)) {
        $line = fgets($file);
        echo $line . "<br>";
    }
    fclose($file);
}
```

## ℹ️ Klasy i obiekty

Klasy to wzorce opisujące obiekty. Obiekt to konkretna instancja klasy, która przechowuje dane tj. `właściwości` i zachowanie tj. `metody`.

Konstruktor `__construct()` jest wywoływany automatycznie podczas tworzenia obiektu i służy do ustawienia początkowych `właściwości`. Destruktor `__destruct()` jest wywoływany, gdy obiekt przestaje istnieć lub skrypt się kończy, co pozwala zwolnić zasoby lub wykonać finalizację.

> Klasa to szablon, a obiekt to z niego stworzona instancja w pamięci.

> Można wyobrazić sobię klase jako foremkę do ciasta, a ciasto jako obiekt który powstał dzięki tej foremce.

#### Przykłady:
```php
<?php
class Car {
    public $brand;
    public $year;

    public function __construct($brand, $year) {
        $this->brand = $brand;
        $this->year = $year;
        echo "Tworzę samochód $brand\n";
    }

    public function __destruct() {
        echo "Usuwam obiekt Car\n";
    }
}

$car = new Car("Toyota", 2020);
echo $car->brand; // Toyota
```

## ℹ️ Metody

Metody to funkcje wewnątrz klas, które wykonują działania na danych obiektu. Metody mogą przyjmować argumenty i zwracać wartości.

> Metody opisują, co obiekt potrafi zrobić - jest to funkcja, ale przeznaczona tylko dla określonej klasy.

Metoda statyczna (`static`) nie działa na konkretnym obiekcie, lecz na klasie. Można ją wywołać bez tworzenia instancji, np. do operacji pomocniczych lub fabryk.

#### Przykłady:
```php
<?php
class Calculator {
    public function add($a, $b) {
        return $a + $b;
    }

    public function describe() {
        return "Prosty kalkulator";
    }

    public static function square($x) {
        return $x * $x;
    }
}

$calc = new Calculator();
echo $calc->add(5, 7); // 12
echo $calc->describe(); // Prosty kalkulator

echo Calculator::square(4); // 16
```

## ℹ️ Modyfikatory dostępu

Dostęp do właściwości i metod klasy można ograniczyć za pomocą modyfikatorów.

> Kontrola dostępu to sposób na ukrycie wewnętrznej implementacji klasy.

| Modyfikator | Dostępność | Opis |
|-------------|------------|------|
| `public` | wszędzie | dostępna z zewnątrz klasy |
| `protected` | klasa i pochodne | dostępna tylko w klasie i klasach dziedziczących |
| `private` | tylko klasa | dostępna tylko wewnątrz tej klasy |

#### Przykłady:
```php
<?php
class User {
    public $name;
    protected $password;
    private $token;

    public function __construct($name, $password) {
        $this->name = $name;
        $this->password = $password;
        $this->token = bin2hex(random_bytes(8));
    }

    public function getToken() {
        return $this->token;
    }
}

$user = new User("Marta", "secret");
echo $user->name; // działa
// echo $user->password; // błąd
// echo $user->token; // błąd
echo $user->getToken();
```

## ℹ️ Dziedziczenie

Dziedziczenie pozwala tworzyć nowe klasy na podstawie istniejących. Klasa potomna dziedziczy właściwości i metody klasy bazowej.

> Dziedziczenie upraszcza kod i umożliwia ponowne użycie wspólnych elementów.

#### Przykłady:
```php
<?php
class Animal {
    public $name;

    public function __construct($name) {
        $this->name = $name;
    }

    public function speak() {
        return "...";
    }
}

class Dog extends Animal {
    public function speak() {
        return "Hau hau";
    }
}

$dog = new Dog("Reksio");
echo $dog->name;   // Reksio
echo $dog->speak(); // Hau hau
```

## ℹ️ Interfejsy

Interfejsy definiują zestaw metod, które każda klasa implementująca dany interfejs musi posiadać. Dzięki temu klasy mogą działać według wspólnego kontraktu, nawet jeśli mają różną implementację.

> Interfejsy mówią, jakie metody muszą istnieć, ale nie zawierają ich implementacji.

#### Przykłady:
```php
<?php
interface Logger {
    public function log(string $message);
}

class FileLogger implements Logger {
    public function log(string $message) {
        file_put_contents('log.txt', $message . "\n", FILE_APPEND);
    }
}

class DatabaseLogger implements Logger {
    public function log(string $message) {
        // zapis do bazy danych, tutaj tylko przykład
        echo "Zapis do bazy: $message";
    }
}

$logger = new FileLogger();
$logger->log("Test");
```

## ℹ️ Abstrakcja

Abstrakcja pozwala definiować klasy lub metody, które muszą zostać zaimplementowane w klasach potomnych. Klasy abstrakcyjne same nie mogą być instancjonowane.

> Abstrakcja opisuje, co klasa powinna robić, ale nie mówi jak dokładnie to zrobić.

#### Przykłady:
```php
<?php
abstract class Shape {
    protected $name;

    public function __construct($name) {
        $this->name = $name;
    }

    abstract public function area();

    public function describe() {
        return "To figura geometryczna: " . $this->name;
    }
}

class Rectangle extends Shape {
    private $width;
    private $height;

    public function __construct($name, $width, $height) {
        parent::__construct($name);
        $this->width = $width;
        $this->height = $height;
    }

    public function area() {
        return $this->width * $this->height;
    }
}

$rect = new Rectangle("prostokąt", 5, 4);
echo $rect->describe(); // To figura geometryczna: prostokąt
echo $rect->area(); // 20
```

- `interface` — inny sposób definiowania kontraktu metod
- Klasy abstrakcyjne mogą mieć zwykłe metody i właściwości
- Klasa abstrakcyjna może zawierać implementację i właściwości, interfejs tylko deklaracje metod.
- Klasa może implementować wiele interfejsów, ale dziedziczyć tylko po jednej klasie abstrakcyjnej.
